Not how much Claude quota you've used — how fast you're burning it.

Every Claude usage monitor tells you the level: 76% used. Burndown Bar tells you the slope. 76% on its own means nothing. 76% with only 69% of the week elapsed means you run dry Friday night and the reset isn't until Saturday afternoon — and that is the sentence your menu bar shows at a glance.

What it does

Reads the trajectory, not the number
For each window — weekly, 5-hour, per-model — it works out your burn rate, your pace against the sustainable rate, and either the headroom you'll land with at reset or the hour you run dry and how far that falls before the reset.
A smart projection that knows your rhythm
A straight-line projection assumes you burn through the night and the weekend — you don't. The weekly window's smart projection walks the hours left to reset and applies your own learned burn for each weekday and hour, so a busy afternoon isn't mistaken for a runaway week. It's the figure the menu bar leads with; the plain projection stays in the dropdown beside it. Thin on history? It falls back to your recent pace and says so.
Trends against your own usual
It quietly records a snapshot each run and learns your rhythm — what a typical Tuesday afternoon looks like, apart from a typical Saturday morning — then reports the present against it: arrows in the menu bar and a two-sentence plain-English summary up top. No model, no API; just your history and arithmetic. A trend is only called when the rate is genuinely out of line with your own variance, so ordinary noise stays quiet.
Notifications when something actually shifts
A heavy day, a spend spike, a balance about to run dry. Each fires once, with a cooldown, and nothing arrives between 22:00 and 08:00.
Catches it when Anthropic moves your limits
Early resets and quiet mid-window quota bumps are detected from the usage stream, logged in the dropdown, and — importantly — left out of your learned baseline, so a holiday grant never becomes your new normal.
OpenRouter credits, optionally
Add a key and it tracks your prepaid balance and extrapolates when it hits zero, surviving top-ups because it reads cumulative spend.
Private, and one file
Stdlib-only Python, no dependencies, auditable in five minutes. Nothing leaves your machine but the same usage request Claude Code already makes.

Install

Requires macOS, SwiftBar, and a Claude Pro or Max subscription with Claude Code logged in.

brew install swiftbar   # if you don't have it yet
curl -fsSL https://raw.githubusercontent.com/alcides-collective/burndown-bar/main/burndown-bar.5m.py \
  -o "$(defaults read com.ameba.SwiftBar PluginDirectory)/burndown-bar.5m.py" \
  && chmod +x "$(defaults read com.ameba.SwiftBar PluginDirectory)/burndown-bar.5m.py"

The first run shows one macOS Keychain prompt — the plugin reads your Claude Code token via the system security tool. Click Always Allow once and you're done. Trends say “still building history” until enough snapshots accumulate; hour and day comparisons appear within a day, week-over-week after about two weeks.
